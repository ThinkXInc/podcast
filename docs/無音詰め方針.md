# 無音（長すぎる空白）詰めの方針

最終処理として、切り出し動画の「無駄に長い空白」だけを詰める。判断基準をここにまとめる。
CLAUDE.md 役割4 はこのファイルを参照する。スクリプト: scripts/detect_silence.py（検出）/
scripts/apply_trim.py（適用）。

## 大原則
1. 基本は詰めない。通しで喋っている自然な感じを最優先する。小〜中の間（息継ぎ・考え・
   転換の間）は味なので残す。細かく詰めると不自然に忙しくなる。
2. 視聴者が離脱するほど長い空白だけを詰める対象にする（目安 4 秒以上。ただし秒数は絶対基準
   ではなく、下記の内容判断を必ず併せる）。
3. 音響解析（ffmpeg silencedetect）は使わない。環境音・ホットな録音ではノイズフロアが高く、
   真の無音を取りこぼしたり発話を無音と誤判定する。判断は必ず文字起こしのタイムスタンプ
   （transcript.json の単語間ギャップ）で行う。

## 無音か「取りこぼし」かは内容の連続性で判断する
長いギャップには 2 種類ある。文字起こしの前後を読んで見分ける（音は要らない）:
- 真の無音（詰めてよい）: 直前で話が一旦切れ（文末・話題の区切り）、直後が新しい文で始まる。
  内容が途切れている。→ 長ければ詰め候補。
- 文字起こしの取りこぼし（詰めない）: 内容が途切れず続いている（直前が節の途中、直後がその
  続き）のに数秒空いている。喋っているのに WhisperX が単語を落とした区間。ここを詰めると
  実際の発話を削ってしまう。→ 長くても触らない。

## どうしても迷うときはユーザーに聞く
真の無音か取りこぼしか判断がつかない長いギャップは、その箇所の前後の文字起こしを提示して
ユーザーに確認する。ユーザーは収録を覚えていることがある（例: そこは席を立ったから詰めていい /
そこは笑っていただけだから残して）。勝手に詰めない・残さない。

## flag（detect_silence.py が付ける補助ラベル。最終判断は内容を読んで上書き可）
- keep_natural   … 自然な間（既定 4 秒未満は全部これ）。残す。
- likely_dropped … 長い＋内容が続く＝取りこぼし疑い。触らない。
- review         … 長い＋内容が切れている＝真の長い無音かも。詰め候補。迷うなら聞く。

## 手順
1. python scripts/detect_silence.py <ID>
   transcript.json＋segments.json から各セグメントの保持区間内の単語間ギャップを
   final.mp4 基準の時刻で拾い、前後の内容と flag を silences.json に出す。
2. Claude が review のギャップだけ内容で最終判断する。詰めると決めたものは、そのセグメントの
   data/<ID>/trim_plan.json に書く（確認は取らず自動で適用。ただし迷う箇所だけ聞く）:
   {"keeps":[{"start_sec":..,"end_sec":..,"keep":0.4,"reason":"…"}, ...]}
   - keep は「その空白に残す秒数」。文の切れ目 0.4〜0.5 / 露骨な無駄間 0.2 前後。詰めすぎない。
   - start_sec / end_sec は final.mp4 基準（detect_silence.py の出力そのまま）。
3. python scripts/apply_trim.py <ID> <そのセグメントの final.mp4> を実行。
   セグメントごとに 1〜3 を回す（trim_plan.json は ID 単位の単一ファイルなので上書き運用）。
4. 元ファイルは必ず残る（final_orig.mp4）。処理後は final_trimmed.mp4。
5. 完了後、各区間をなぜそうしたか自然言語でまとめて説明する（数字の羅列ではなく理由）。
   気になる区間があれば keep を直して再実行できることを一言添える。

## やらないこと
- 一律閾値で機械的に全部詰める（不自然・取りこぼし削りの事故）。
- 音響解析での無音判定。
- 迷ったまま黙って詰める・黙って残す（→ ユーザーに聞く）。
