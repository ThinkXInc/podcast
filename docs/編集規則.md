# 編集規則（切り出し・カットの一般規則）

このPodcastは **大塚さんの発言を集める**コンテンツ。切り出し・カットは以下の一般規則に従う。
個別作業の手順は `CLAUDE.md`、無音詰めの詳細は `docs/無音詰め方針.md` を参照。

---

## 最重要の2原則

1. **Speaker1（大塚さん）の発言は、ほぼそのまま残す。**
   - 主役は大塚さんの論。安易にカットしない。迷ったら残す。
   - GPT(suggest)が付ける `cuts` は**範囲が広すぎて大塚さんの本編を巻き込む**ことが多い。
     固有名や独り言を消すつもりで、直前・直後の核心発言まで削っていないか必ず確認する。
     → 広すぎるカットは**範囲を狭める**か**復活**させる（例: 「山田さんのc」を消すために
       その手前50秒の技術論まで削っていた／「メディア＝インターフェース」核心を
       「手元確認の独り言」カットが巻き込んでいた）。

2. **会話相手（Speaker≠1）の発言は、基本カットする。**
   - 相手の相槌・質問・意見はなるべく出さない。視聴者に「もたつき感」を与えるため。
   - 例外: 相手の質問が大塚さんの回答の前振りとして必須で、切ると回答が宙に浮く場合。
     その箇所だけ個別に確認する。

## 話者情報の出所

- WhisperX の `transcript.json` には**話者ラベルが無い**。
- **Notta の全文txt**（`data/<ID>/*-transcript*.txt`）に `HH:MM:SS Speaker N` 形式である。
  時刻は音源と**同一タイムライン**（segments.json の start_sec と揃う）。
- **Speaker 1 = 大塚さん**（最多話者）。Speaker 2〜 が会話相手。
- 運用: 各セグメントの Speaker≠1 区間を洗い出し、drops に入れる。
- **注意: Notta の話者境界は WhisperX の単語時刻と数秒ズレることがある**。
  会話相手のターン末尾が Notta で早く切られ、大塚さん側に混ざって見えることがある
  （例: 民主主義の会2-5 ⑥ 1:40:14 付近。「僕がまだ腹落ちしていない」は会話相手の発言だが
  Notta は 1:40:14 を Speaker1 と誤ラベル）。**最終判断はユーザーの記憶／WhisperX本文で上書き**し、
  drops は WhisperX 時刻（＝render が切る時間軸）で指定する。

## drops の考え方（再構築の指針）

確定カット `drops` は原則、次の合算で構成する（それ以外の Speaker1 発言は残す）:

```
drops = 会話相手(Speaker≠1)の全ブロック  +  承認済みの手動カット
```

- **復活は安全方向**（実発話を戻すだけ）なので、規則に沿う復活は積極的に行う。
- **カットは1件ずつ承認を取ってから** drops に入れる（CLAUDE.md の原則）。

## カットしてよい類型（大塚さんの発言でも可。ただし要承認・範囲は最小限）

- **個人名・内輪参照**（例: 「山田さんのcの話」）。前後の本編は残し、固有名の一言だけ。
- **手元確認・独り言**（例: 「何のアプリ入ってたかな」等のフィラー）。
- **明確な脱線**（例: 食事の話・収録進行のメタ「前回の復習」「逸れてきてる」）。
  ただし脱線の**末尾が本編に復帰**していることが多いので、脱線部だけを最小で切る。
- **過度に批判を煽る断定**（例: 「教育としても研究としても終わってる」）。
- **事実誤認・公開リスク**（数値の断定、政治的断定、民族・実在人物への断定等）。
  勝手に消さず「カット/言い換え」を確認する（suggest の `cuts`/`fact_checks` 参照）。

いずれも**カット範囲は最小限**。核心発言を巻き込まないこと（原則1）。

## 無音詰め（最後の工程）

- 全カット確定後、最後に無音を詰める。判定は**文字起こしの単語間ギャップ**
  （`silences.json`。音響解析はしない）。詳細は `docs/無音詰め方針.md`。
- **`likely_dropped`（内容が続く長い空白＝文字起こしの取りこぼし）は触らない**（実発話を消さない）。
- 強さのパターン（比較して選ぶ。ハードは不採用）:
  - **自然（採用基準）**: 1.5秒以上の間を 0.4秒へ。息継ぎは残しつつ間延びを解消。
  - **ほぼそのまま**: 3.5秒以上だけ 0.8秒へ。明らかな長い空白のみ。
- 適用は `apply_trim.py`。元は必ず `*_orig.mp4` に残す。
